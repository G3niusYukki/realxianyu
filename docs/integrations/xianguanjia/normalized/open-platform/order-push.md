# 闲管家开放平台 - 订单推送通知（归一化）

## 1. 用途
闲管家在订单状态变更时，向商家配置的 URL 推送订单快照，用于订单同步、履约追踪、售后处理。

## 2. 请求路径方法
- **Type**: Webhook（闲管家主动 POST 到商家配置的推送地址）
- **Method**: `POST`
- **Content-Type**: `application/json`
- **超时**: 3 秒
- **重试**: 最多重试 3 次

## 3. 请求字段（闲管家 → 商家）
| 字段名 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| seller_id | string | 否 | 卖家ID（商务对接） | - |
| user_name | string | 否 | 店铺会员名 | - |
| order_no | string | 是 | 闲鱼订单号 | 3364202298717566229 |
| order_type | int | 是 | 订单类型（1-10） | 1 |
| order_status | int | 是 | 订单状态：11/12/21/22/23/24 | 22 |
| refund_status | int | 是 | 退款状态（0-8） | 0 |
| modify_time | int | 是 | 修改时间（时间戳秒） | 1685087039 |
| product_id | int64 | 否 | 管家商品 ID | - |
| item_id | int64 | 否 | 闲鱼 item_id | - |

## 4. 返回字段（商家 → 闲管家）
商家需在 **3 秒内** 返回 JSON 响应（建议先入队再异步处理）：

| 字段名 | 类型 | 说明 | 示例 |
|---|---|---|---|
| result | string | 固定值 `success` 表示接收成功 | success |
| msg | string | 提示信息 | 接收成功 |

示例：`{"result": "success", "msg": "接收成功"}`

## 5. 错误码
| 场景 | 含义 | 处理建议 |
|---|---|---|
| 超时（3 秒） | 闲管家会重试，最多 3 次 | 商家务必快速返回，业务逻辑异步处理 |
| 返回非 success | 可能触发重试 | 确保接收成功后再返回 success |

## 6. 签名规则
- 是否需要签名：按闲管家文档配置；若支持签名校验，商家需验证请求来源。
- Base URL：商家自行配置推送地址，无固定 Base URL。
- 密钥说明：若启用签名，需从闲管家开放平台获取并妥善保管。

## 7. 幂等建议
- 以 `order_no + modify_time` 作为幂等键。
- 重复推送时返回相同成功响应，避免闲管家重复重试。
- 业务处理端根据幂等键去重后落库或更新。

## 8. 重试建议
- **闲管家侧**：3 秒超时，最多重试 3 次。
- **商家侧**：收到推送后**立即返回 success**，将数据写入队列或缓存，异步处理订单业务逻辑。
- 避免在回调内执行耗时操作（如查库、外部调用），否则易超时并触发闲管家重试，造成重复推送。
- 订单业务处理失败时走内部重试或死信队列，并告警人工介入。
