# 闲管家开放平台 - 商品回调通知（归一化）

## 1. 用途
闲管家在商品上架、编辑等任务完成后，向商家配置的 URL 推送回调通知，用于异步结果同步、状态追踪与业务对账。

## 2. 请求路径方法
- **Type**: Webhook（闲管家主动 POST 到商家配置的回调地址）
- **Method**: `POST`
- **Content-Type**: `application/json`

## 3. 请求字段（闲管家 → 商家）
| 字段名 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| seller_id | string | 否 | 卖家ID（商务对接） | - |
| product_id | int64 | 是 | 管家商品 ID | 123456 |
| product_status | int | 是 | 商品状态：-1/21/22/23/31/33/36 | 21 |
| publish_status | int | 是 | 发布状态：-1/1/2/3/4/5/9 | 1 |
| item_biz_type | int | 是 | 商品业务类型：2/0/10/16/19/24/26/35 | 2 |
| user_name | string | 否 | 店铺会员名 | - |
| item_id | int64 | 否 | 闲鱼 item_id | - |
| task_type | int | 是 | 任务类型：10=上架新商品，11=上架存量，13=编辑在架 | 10 |
| task_time | int | 是 | 任务时间（时间戳秒） | 1685087039 |
| task_result | int | 是 | 任务结果：1=成功，2=失败 | 1 |
| err_code | string | 否 | 错误码（task_result=2 时） | - |
| err_msg | string | 否 | 错误描述（task_result=2 时） | - |

## 4. 返回字段（商家 → 闲管家）
商家需在合理时间内返回 JSON 响应，表示已接收回调：

| 字段名 | 类型 | 说明 | 示例 |
|---|---|---|---|
| result | string | 固定值 `success` 表示接收成功 | success |
| msg | string | 提示信息 | 接收成功 |

示例：`{"result": "success", "msg": "接收成功"}`

## 5. 错误码
| 场景 | 含义 | 处理建议 |
|---|---|---|
| 未按时返回 | 闲管家可能重试 | 建议快速入队并异步处理，先返回成功 |
| 返回非 success | 可能触发重试 | 确保业务处理成功后再返回 success |

## 6. 签名规则
- 是否需要签名：按闲管家文档配置；若支持签名校验，商家需验证请求来源。
- Base URL：商家自行配置回调地址，无固定 Base URL。
- 密钥说明：若启用签名，需从闲管家开放平台获取并妥善保管。

## 7. 幂等建议
- 以 `product_id + task_type + task_time` 或 `product_id + item_id + task_type` 作为幂等键。
- 重复推送时返回相同成功响应，避免闲管家重复重试。
- 业务处理端根据幂等键去重后落库。

## 8. 重试建议
- 闲管家侧：通常会有重试机制，商家需能处理重复推送。
- 商家侧：收到回调后应快速入队、立即返回 success，异步处理业务逻辑。
- 业务处理失败时走内部重试或死信队列，避免超时导致闲管家频繁重试。
