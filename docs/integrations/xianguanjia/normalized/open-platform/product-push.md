# 闲管家开放平台 - 商品推送通知（归一化）

## 1. 用途
闲管家在商品信息变更时，向商家配置的 URL 推送商品快照，用于商品同步、库存/价格更新、状态监控。

## 2. 请求路径方法
- **Type**: Webhook（闲管家主动 POST 到商家配置的推送地址）
- **Method**: `POST`
- **Content-Type**: `application/json`

## 3. 请求字段（闲管家 → 商家）
| 字段名 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| seller_id | string | 否 | 卖家ID（商务对接） | - |
| product_id | int64 | 是 | 管家商品 ID | 123456 |
| product_status | int | 是 | 商品状态 | - |
| publish_status | int | 是 | 发布状态 | - |
| item_biz_type | int | 是 | 商品业务类型 | - |
| price | int | 是 | 商品价格（单位：分，范围 1-9999999900） | 9900 |
| stock | int | 是 | 库存（范围 1-399960） | 100 |
| user_name | string | 否 | 店铺会员名 | - |
| modify_time | int | 是 | 修改时间（时间戳秒） | 1685087039 |

## 4. 返回字段（商家 → 闲管家）
商家需在合理时间内返回 JSON 响应：

| 字段名 | 类型 | 说明 | 示例 |
|---|---|---|---|
| result | string | 固定值 `success` 表示接收成功 | success |
| msg | string | 提示信息 | 接收成功 |

示例：`{"result": "success", "msg": "接收成功"}`

## 5. 错误码
| 场景 | 含义 | 处理建议 |
|---|---|---|
| 未按时返回 | 闲管家可能重试 | 建议快速入队并异步处理，先返回 success |
| 返回非 success | 可能触发重试 | 确保业务处理成功后再返回 success |

## 6. 签名规则
- 是否需要签名：按闲管家文档配置；若支持签名校验，商家需验证请求来源。
- Base URL：商家自行配置推送地址，无固定 Base URL。
- 密钥说明：若启用签名，需从闲管家开放平台获取并妥善保管。

## 7. 幂等建议
- 以 `product_id + modify_time` 或 `product_id + user_name + modify_time` 作为幂等键。
- 重复推送时返回相同成功响应。
- 业务处理端根据幂等键去重后更新本地商品信息。

## 8. 重试建议
- 闲管家侧：通常会有重试机制，商家需能处理重复推送。
- 商家侧：收到推送后应快速入队、立即返回 success，异步处理业务逻辑。
- 商品同步失败时走内部重试或告警，避免超时导致闲管家频繁重试。
