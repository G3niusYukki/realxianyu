## 一、闲管家配置面板（Dashboard.tsx — 唯一新增 UI）

### 目标
在 Dashboard 首页添加闲管家可视化配置卡片，与现有系统状态卡片并列。

### 位置
`client/src/pages/Dashboard.tsx`，插入在状态监控区（`ApiStatusPanel`）之后。

### UI 结构
```tsx
// 状态
const [xgjAppKey, setXgjAppKey] = useState('');
const [xgjAppSecret, setXgjAppSecret] = useState('');
const [xgjAutoPrice, setXgjAutoPrice] = useState(false);
const [xgjAutoShip, setXgjAutoShip] = useState(false);
const [xgjConnected, setXgjConnected] = useState<boolean | null>(null);
const [xgjSaving, setXgjSaving] = useState(false);

// 保存
const handleSaveXgj = async () => {
  setXgjSaving(true);
  await api.post('/xgj/settings', {
    app_key: xgjAppKey,
    app_secret: xgjAppSecret,
    auto_price_enabled: xgjAutoPrice,
    auto_ship_enabled: xgjAutoShip,
  });
  setXgjSaving(false);
};

// 测试连接
const handleTestConnection = async () => {
  const res = await api.post('/xgj/test-connection', {
    app_key: xgjAppKey,
    app_secret: xgjAppSecret,
    mode: 'xianguanjia',  // 闲管家模式
  });
  setXgjConnected(res.data?.ok ?? false);
};
```

**卡片 UI（~80行tsx）：**
- 连接状态 Badge（未配置 / 连接中 / 成功 / 失败）
- AppKey 输入框
- AppSecret 密码输入框
- 自动改价开关 + 支付后自动发货开关（左右排列）
- 测试连接按钮（左侧）+ 保存按钮（右侧 primary）
- 重试状态行（改价重试次数 / 发货重试次数，如 >0 则显示）

### 重要：字段名必须为 snake_case
| 前端字段 | 后端期望 |
|---|---|
| `app_key` | ✅ |
| `app_secret` | ✅ |
| `auto_price_enabled` | ✅ |
| `auto_ship_enabled` | ✅ |

### API 调用（后端已实现）
- `POST /api/xgj/settings` — 保存配置
- `POST /api/xgj/test-connection` — 测试连接（含 `mode: 'xianguanjia'`）
- `POST /api/xgj/retry-price` — 手动重试改价
- `POST /api/xgj/retry-ship` — 手动重试发货

## 二、rule_suggestions.py（✅ 已存在，确认功能完整）

`src/dashboard/routes/rule_suggestions.py`（222行）已完整实现。

### 端点

#### `POST /api/rule-suggestions/generate`
- 读取 `data/unmatched_messages.jsonl` 最多 100 条（精确字符串去重）
- 读取现有规则名称避免重复
- 调用 `ContentService._call_ai()` 生成 5 条规则建议
- 返回: `{ suggestions: [{ keywords, reply, priority, reason }, ...] }`

#### `POST /api/rule-suggestions/apply`
- 注意：请求体为**单个** `rule` 对象（非 `rules` 数组）
- 追加到 `custom_intent_rules`
- 写入 `data/system_config.json`
- 调用 `get_config().reload()` 热重载
- 返回: `{ success: true, count: 1 }`

### 现状
- 后端已完整实现
- 前端 `client/src/api/dashboard.ts` 第 117-127 行 API 绑定已存在
- 前端 `client/src/pages/Dashboard.tsx` 第 526-576 行规则建议卡片已存在

**本次无需代码改动。**
